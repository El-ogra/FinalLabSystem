# مواصفات نافذة إدخال نتائج التحاليل — النظام المرجعي (Real Lab System)

> هذا الملف مستخرج من وثائق النظام المرجعي (ملف CHM + ملف PDF التعليمي).  
> يُستخدم كمرجع مقارنة للتحقق من مطابقة تنفيذ FinalLabSystem لهذه النافذة.

---

## 1. طريقة الوصول للنافذة

- من الشريط العلوي: زر **"المرضى"** ← ثم زر **"إدخال نتائج التحاليل"**
- اختصار لوحة المفاتيح: **F4** من أي نافذة رئيسية في البرنامج

---

## 2. التخطيط العام للنافذة (Layout)

النافذة مقسّمة إلى **ثمانية أقسام وظيفية** واضحة:

```
┌─────────────────────────────────────────────┬──────────────────────┐
│  [2] رأس بيانات المريض (Header)             │  [1] قائمة المرضى   │
├─────────────────────────────────────────────│      (Patient List)  │
│  [3] شبكة التحاليل (Tests Grid)             │                      │
│                                             │  [7] لوحة الفلاتر   │
│                                             │      والتصنيف       │
├─────────────────────────────────────────────│                      │
│  [4] أزرار الحالة اليدوية + [8] البحث      │  Refresh / Icons     │
├─────────────────────────────────────────────┴──────────────────────┤
│  [5] شريط الأدوات السفلي (Bottom Toolbar)                         │
└────────────────────────────────────────────────────────────────────┘
```

---

## 3. القسم (1) — قائمة مرضى اليوم (Patient List Panel)

**الموقع:** الجانب الأيمن من النافذة  
**الوظيفة:** يعرض مرضى اليوم الحالي فقط بشكل افتراضي

### أيقونات حالة المريض
بجوار كل اسم مريض أيقونة توضح حالته الحالية:

| الأيقونة | المعنى |
|---|---|
| 🔴 دائرة حمراء (NEW) | مريض جديد — لم تُدخل له أي نتائج بعد |
| 📋 ورقة ملاحظات | مريض له نتائج لم تُكتب بعد |
| 🔄 سهمان (أخضر وأزرق) | مريض له نتائج لم تُراجَع بعد |
| 🖨️ طابعة | مريض له نتائج لم تُطبَع بعد |
| 🛒 عربة تسوق | مريض لم يستلم نتائجه بعد |
| 💷 جنيه | مريض استلم نتائجه لكن يوجد مشكلة في حسابه (باقي مبلغ) |
| 🏅 وسام | مريض مكتمل — نتائجه مُدخلة ومطبوعة وحسابه خالص واستلم نتائجه |

### السلوك:
- الضغط على اسم المريض ← يعرض بياناته وتحاليله في الأقسام المجاورة
- التنقل بالأسهم (Up/Down) بين المرضى

---

## 4. القسم (2) — رأس بيانات المريض (Patient Info Header)

**الموقع:** أعلى يسار النافذة

### الحقول المعروضة:
- **الاسم** (Name)
- **النوع** — Male / Female (النوع / الجنس)
- **العمر** (Age / العمر)
- **الكود** — كود المريض في النظام
- **Lab ID** — كود المعمل للزيارة الحالية
- **صورة المريض** (Photo) — أيقونة على اليمين
- **خانة صغيرة خضراء** بها عدد مرات زيارة هذا المريض للمعمل
- في حال المريض VIP/مهم: خانة الاسم تتميّز **باللون الأحمر**

### ملاحظة خاصة:
- يمكن نسخ كود المريض أو Lab ID بضغطة واحدة على الخانة

---

## 5. القسم (3) — شبكة التحاليل (Tests Grid)

**الموقع:** المنطقة الوسطى الرئيسية

### أعمدة الشبكة:
| العمود | الوصف |
|---|---|
| **Test Abbreviation** | اسم التحليل المختصر |
| **[P] [T]** | زرّان للـ Audit Trail (راجع القسم 6) |
| **Result** | نتيجة التحليل — إما قيمة مُدخلة أو "See Report" للتحاليل ذات المكونات المتعددة |
| **Status** | حالة التحليل |
| **Finish** | ✅ تم الانتهاء |
| **Verify** | ✅ تمت المراجعة |
| **Print** | ✅ تمت الطباعة |
| **Export** | ✅ تم التسليم |

### ألوان الصفوف:
- التحاليل ذات المكونات المتعددة (See Report): تظهر بلون مختلف (برتقالي/أخضر)
- التحاليل البسيطة ذات قيمة مُدخلة: تظهر بلون مختلف

### كيفية إدخال النتيجة:
- **نقر مزدوج** على التحليل، أو
- تحديده بالأسهم ثم **Enter**
- ← تفتح نافذة إدخال النتيجة الخاصة بذلك التحليل

### التحاليل ذات المكونات المتعددة (See Report):
- تُظهر "**See Report**" في عمود Result بدلاً من قيمة مباشرة
- عند النقر المزدوج عليها ← **تفتح نافذة مستقلة مخصصة** (راجع القسم 9)
- لا تُدخَل كسطر عادي في الشبكة

---

## 6. القسم (4) — أزرار الحالة اليدوية

### زر "تمت" (Manual Override):
- **الغرض:** لتحاليل تم إنجازها على أجهزة خارجية (جهاز صور الدم، جهاز السائل المنوي...) ولم تُدخَل نتيجتها من النظام
- الضغط عليه ← يعتبر النظام أن التحليل تم/طُبع/رُوجع
- الضغط مرة أخرى ← يعود للحالة غير المكتملة (toggle)

### زر "ملاحظات":
- إضافة أو تعديل ملاحظة لباقي الأطباء أو موظفي الاستقبال عن هذا المريض

---

## 7. القسم (5) — شريط الأدوات السفلي (Bottom Toolbar)

الأزرار من اليمين إلى اليسار:

| الزر | الوظيفة |
|---|---|
| **بيانات المريض** | الانتقال لنافذة بيانات المريض |
| **تقرير مجمع** | طباعة تقرير مجمع للنتائج المنتهية والمراجعة (راجع القسم 10) |
| **معاينة** | معاينة التقرير قبل الطباعة |
| **طباعة** | طباعة التقرير مباشرة |
| **تاريخ المرضي** | طباعة التاريخ المرضي أو عمل تاريخ مرضي مخصص |
| **تقرير فارغ** | طباعة تقرير فارغ به الباركود واسم وشعار المعمل |
| **طباعة طرف** | طباعة ورقة عمل (أمر شغل) للمريض المحدد |
| **SMS** | إرسال النتيجة بالرسالة النصية |
| **القائمة الرئيسية** | العودة للقائمة الرئيسية |

## 8. القسم (6) — Audit Trail (زرّا P و T)

**الموقع:** في رأس الشبكة بجوار "Test Abbreviation"

### زر P (بيانات التسجيل):
يعرض:
- المستخدم الذي سجّل بيانات المريض
- عدد مرات التعديل على بياناته
- آخر مستخدم قام بالتعديل
- نظام دفع الحساب والمستخدمين الذين استلموا المبالغ المالية

### زر T (تتبع النتيجة):
بعد تحديد أي تحليل والضغط على T يعرض:
- من أدخل النتيجة
- من راجعها
- المسؤول عن الطباعة
- عدد مرات طباعة النتيجة
- من سلّم النتيجة
- كل ذلك **بالتاريخ والوقت**

> ⚠️ هذه الخاصية متاحة فقط لـ **مدير النظام** أو أصحاب الصلاحيات الكاملة

---

## 9. القسم (7) — لوحة الفلاتر والتصنيف

**الموقع:** الجانب الأيمن أسفل قائمة المرضى

### تبويبات التصنيف:
| التبويب | المعنى |
|---|---|
| **Individual** | المرضى الأفراد |
| **LabToLab** | مرضى المعامل الأخرى |
| **Contracts** | مرضى الجهات والعقود |
| **VIP** | المرضى المهمون |
| **Free** | المرضى المعفيون |
| **All** | جميع المرضى |

### الوصول السريع بالحالة:
- مرضى لم تُكتب نتائجهم
- مرضى لم تُراجَع نتائجهم
- مرضى لم تُطبَع نتائجهم
- سهمَا التوسعة (الأزرق) ← للوصول لمرضى يوم محدد بإدخال تاريخه

### زر Refresh:
- إعادة تحميل قائمة مرضى اليوم (F5)

---

## 10. القسم (8) — صندوق البحث

**الموقع:** أسفل النافذة

### يدعم البحث بـ:
- كود المريض
- Lab ID (كود المعمل)
- كود الفيل (VIP code)
- الاسم (ضمن مرضى اليوم المحدد فقط)
- رقم الحضور في اليوم (مثل: 5 أو 12)

---

## 11. نافذة التحليل ذي المكونات المتعددة (Multi-Component Test Window)

**تُفتح عند:** النقر المزدوج على أي تحليل يعرض "See Report" في الشبكة

### بنية النافذة:

#### رأس النافذة (Header):
- النوع، الاسم، الكود، جهة الإحالة، كود المعمل، الصورة، العمر

#### عنوان المجموعة:
- اسم مجموعة التحليل (مثلاً: "Coagulation Profile")

#### شبكة المكونات:
| العمود | الوصف |
|---|---|
| **Test Name** | اسم المكوّن |
| **Result** | خانة إدخال النتيجة |
| **Unit** | الوحدة |
| **L or H** | مؤشر منخفض/مرتفع |
| **Normal Ranges** | المدى الطبيعي |
| **Verified** | ✅ مراجعة |
| **Print** | ✅ طباعة |

#### القيم الثابتة (Constants Panel):
- لوحة جانبية على اليمين تعرض قيماً مرجعية ثابتة مرتبطة بالتحليل (مثل: ISI = 1.5 لـ PT)

#### خانة التعليق (Comment):
- حقل نصي متعدد الأسطر لإضافة تعليق على النتيجة
- مع checkboxes للتحكم في طباعة التعليق

#### أزرار النافذة:
| الزر | الوظيفة |
|---|---|
| **حفظ** | حفظ النتائج المُدخلة |
| **طباعة** | طباعة التقرير |
| **معاينة الطباعة** | معاينة قبل الطباعة |
| **Patient History** | عرض التاريخ المرضي |
| **رجوع** | العودة للنافذة الرئيسية |
| **القائمة الرئيسية** | العودة للقائمة الرئيسية |

### أمثلة على تحاليل متعددة المكونات:
- **Coagulation Profile:** Bleeding، Clotting، PT (Patient's time + Concentration + INR + Ratio)، APTT، Fibrinogen، FDP
- **Stool، Urine، CBC، ESR، Widal Test، Blood Culture** — كلها "See Report"

## 12. نافذة التقرير المجمع (Combined Report Window)

**تُفتح بالضغط على:** زر "تقرير مجمع" في شريط الأدوات

### بنية النافذة:
- **العمود الأيسر "Patient tests in our lab":** قائمة بجميع تحاليل المريض (مع عددها)
- **العمود الأيمن "Available tests in report":** التحاليل المضافة للتقرير المجمع (مع عددها)
- **أزرار النقل:** أسهم لنقل التحاليل بين القائمتين (إضافة/إزالة)
- **ترتيب ظهور التحاليل:** بالضغط على التحليل وتحريكه بالأسهم
- **Checkboxes خيارات:**
  - Print sub title (group name) in report
  - Print printed test again without msg

### أزرار النافذة:
- **القائمة الرئيسية، رجوع، Patient History، معاينة الطباعة، طباعة**

### مثال على تقرير مجمع مطبوع:
يجمع في صفحة واحدة تحاليل من مجموعات متعددة:
- LIVER FUNCTIONS: SGPT، SGOT، AST/ALT Ratio
- Erythrocyte Sedimentation Rate: First Hour
- Coagulation Profile: Control time، PT، Concentration، INR

---

## 13. نافذة التقرير الفارغ (Any Report / Empty Report)

**الغرض:** إدخال نتائج تحاليل يدوياً بدون ربطها بنوع تحليل محدد

### خصائص:
- Page 1 / Page 2 تبويبان
- أعمدة: Test Name، Result، Unit، L or H، Reference Ranges
- خانة "Report Title" في الأعلى
- حقل تعليق (Comment)
- زر "معاينة و طباعة التقرير"
- زر "حفظ"، زر "رجوع"

---

## 14. نافذة نتائج الزراعة (Blood Culture Window)

**تُفتح بالضغط على:** التحليل "Blood Culture" في الشبكة

### تبويبان:
1. **Culture** — بيانات الزراعة:
   - Sample، Organism A، Organism B، Organism C
   - Culture Condition
   - Colony Count
   
2. **Sensitivity** — حساسية المضادات الحيوية:
   - اختيار Organism (A/B/C)
   - أزرار التصنيف: Highly For / Moderate For / Low For / Resistant For
   - قائمة المضادات الحيوية مع: Abbr.، Scientific Name، Sensitivity (mm)
   - إضافة مضاد حيوي: بنقر مزدوج أو تحديد + Add
   - خيارات الإظهار في التقرير: Sensitivity / Reference / Commercial Name

### القسم المجهري (Microscopic):
- Reaction، Pus Cells، R.B.Cs، Epithelial Cells، Crystal، Fungi، Bacteria، Others

### أزرار:
- حفظ، طباعة، معاينة الطباعة، رجوع، تقرير صور، القائمة الرئيسية

## 15. اختصارات لوحة المفاتيح الكاملة

| المفتاح | الوظيفة |
|---|---|
| **F2** | الانتقال لنافذة بيانات المرضى |
| **F3** | الانتقال لنافذة البحث |
| **F4** | الانتقال لنافذة إدخال النتائج |
| **F5** | Refresh — إعادة تحميل قائمة مرضى اليوم |
| **F6** | الانتقال لنافذة تسليم النتائج |
| **F7** | الانتقال لنافذة العينات المرسلة للخارج |
| **F8** | تبديل حالة المراجعة (Verified / Not Verified) |
| **F9** | تبديل حالة الإنجاز (Finished / Not Finished) |
| **F12** | تبديل حالة الطباعة (Printed / Not Printed) |
| **Enter** | عرض تحاليل المريض أو فتح نافذة إدخال النتيجة |
| **↑ ↓** | التنقل بين المرضى / التنقل بين تحاليل المريض |
| **Ctrl** | التبديل بين قائمة المرضى وشبكة التحاليل |
| **Esc** | العودة للقائمة الرئيسية |

## 16. قواعد العمل (Business Rules)

1. **يجب مراجعة النتائج** قبل أن يتمكن المستخدم من طباعتها
2. **يجب طباعة النتائج** قبل أن يتمكن المستخدم من تسليمها للمريض
3. في حال عدم تنفيذ ذلك فعلياً (جهاز خارجي) → يُستخدم زر "تمت" لتجاوز هذا الشرط
4. Audit Trail كامل: كل عملية (إدخال، مراجعة، طباعة، تسليم) مسجلة بالمستخدم والتاريخ والوقت
5. التحاليل ذات المكونات المتعددة تُفتح في **نافذة مستقلة** — لا تُدخَل في شبكة التحاليل الرئيسية مباشرة
